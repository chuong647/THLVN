I/ git
1/ git là gì 
		-git là phần mềm quản lý mã nguồn phân tán
		-phầm mềm mã nguồn mở theo giấy phép công cộng GPLv2
		(The GNU General Public License v2)
		-là một dạng của hệ thông quản lý phiên bản - version control system(VCS)
	*git ra đời năm 2005 do linux tovalds tạo ra
	*mục đích tạo ra git ban đầu là dành cho việc phát triển nhân linux
*hiện nay, git trở thành một trong các phần mềm quản lý mã nguồn phổ biến nhất
2/ tại sao phải sử dụng git
	-git dễ sử dụng, an toàn và nhanh chóng
	-có thể giúp quy trình làm việc nhóm đơn giản hơn rất nhiều bằng việc kết hợp các branch(phân nhánh).
	-bạn có thể làm việc ở bất kỳ đâu vì chỉ cần clone mã nguồn từ kho chứa hoặc clone một phiên bản thay đổi nào đó từ kho chứa, hoặc một nhánh nào đó từ kho chứa.
	-Dễ dàng trong việc deployment sản phẩm.
	-ngoài ra còn có nhiều lợi ích khác.
II/ github
1/ github là gì? 
	-github là một dịch vụ lưu trữ dựa trên web cho các dự án phát triển phẩn mềm sử dụng các hệ thống kiểm soát phiên bản git
2/ github có nhưng loại tài khoản nào?
	-github hiện có 2 loại tài khoản:
		+trả tiền cho repo tư nhân.
		+tài khoản miễn phí dánh cho các dự án mã nguồn mở
3/hiện github có khoản 3,4 tr người dùng và đang trở thành code host lớn nhất trên thế giới
4/ đăng ký tài khoản github như thế nào ?
	bạn có thể tạo tài khoản nhanh chóng và dễ dàng tại đây: https://github.com/join






	-GitHub là một hệ thống quản lý dự án và phiên bản code
	 -hoạt động giống như một mạng xã hội cho lập trình viên.
	-repository và Github chính là một dịch vụ máy chủ repository công cộng.
	 -Github có đầy đủ những tính năng của Git, ngoài ra nó còn bổ sung những tính năng về social để các developer tương tác với nhau.
	-Là công cụ giúp quản lý source code tổ chức theo dạng dữ liệu phân tán.
	-Giúp đồng bộ source code của team lên 1 server.
	-Hỗ trợ các thao tác kiểm tra source code trong quá trình làm việc
